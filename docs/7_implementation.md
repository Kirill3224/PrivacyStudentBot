# 7. План впровадження (Implementation Plan)

Цей документ описує реалістичні кроки для впровадження "Privacy Kit" як практичного стандарту для студентських проєктів в **КАІ**.

План ґрунтується на наших даних: ми вирішуємо проблему **"вакууму контролю"** (**71.4%** викладачів не перевіряють приватність) та **"браку часу"** у розробників (**78.6%**).

---

### Фаза 1: Пілотний запуск та Збір фідбеку (Найближчі 2–4 тижні)

Мета: Перевірити "Kit" на реальних користувачах та зібрати докази для `docs/8_metrics.md`.

| Кроки                                                                                                                                           | Ресурси                            | Відповідальні         | Дедлайн   |
| :---------------------------------------------------------------------------------------------------------------------------------------------- | :--------------------------------- | :-------------------- | :-------- |
| 1. Знайти 3-5 "дружніх" студентських проєктів (через опитування або чати факультету).                                                           | Наш час (комунікація).             | [Ім'я, напр., Богдан] | 1 тиждень |
| 2. Надати їм наш "Kit" (папку `/artifacts`) з проханням "спробувати впровадити".                                                                | `docs/info_pager.md` (інструкція). | [Ваше ім'я]           | 1 тиждень |
| 3. Провести коротке (15 хв) інтерв'ю з кожним пілотним розробником: _Що було незрозуміло?_ _Скільки часу це зайняло?_ (Це дані для Метрики #3). | Шаблон опитування.                 | [Ім'я товариша]       | 3 тижні   |
| 4. Фіналізувати партнерство з командою [Назва команди Анни] та інтегрувати їх посилання в `artifacts/3_security_checklist.md`.                  | Текст від Анни.                    | [Ваше ім'я]           | 2 тижні   |

---

### Фаза 2: Впровадження на рівні факультету (Наступний семестр)

Мета: Перетворити "Privacy Kit" з "проєкту" на "офіційно рекомендований інструмент".

| Кроки                                                                                                                                     | Ресурси                                | Відповідальні   | Дедлайн           |
| :---------------------------------------------------------------------------------------------------------------------------------------- | :------------------------------------- | :-------------- | :---------------- |
| 1. Підготувати Pitch Deck (8 слайдів) на основі **реальних даних** (N=14) та результатів "Фази 1".                                        | `Pitch_Deck.pptx`                      | Вся команда     | 4 тижні           |
| 2. Захистити проєкт перед викладачем та (що важливіше) **презентувати** його **Студраді** та/або **керівництву ФКНТ**.                    | Наші дані (N=14) — це головний ресурс. | [Ваше ім'я]     | Кінець семестру   |
| 3. Запропонувати "Privacy Kit" як **обов'язковий (або рекомендований)** стандарт для всіх курсових/дипломних проєктів, які обробляють ПД. | Підтримка Студради.                    | [Ваше ім'я]     | Кінець семестру   |
| 4. Провести одне `tabletop-тренування інцидентів` (ідея з `Первого черновика`) з адмінами 2-3 сервісів, щоб показати цінність чек-ліста.  | `artifacts/3_security_checklist.md`    | [Ім'я товариша] | Наступний семестр |

---

### Фаза 3: Масштабування та Автоматизація (Наступний рік)

Мета: Забезпечити "життя" проєкту після нашого випуску; зробити його стандартом **КАІ**.

| Кроки                                                                                                                                               | Ресурси                 | Відповідальні | Дедлайн |
| :-------------------------------------------------------------------------------------------------------------------------------------------------- | :---------------------- | :------------ | :------ |
| 1. Передати "Privacy Kit" на баланс Студради або IT-клубу **КАІ** як "стандарт спільноти".                                                          | Студрада.               | "Наступники"  | +1 рік  |
| 2. Розробити **часткову автоматизацію** (ідея з `Первого черновика`), напр., Telegram-бота, який генерує `policy_template.md` на основі 3-х питань. | Наш GitHub репозиторій. | Tech Lead     | +1 рік  |
| 3. UX-полірування (з `Первого черновика`): A/B-тест TL;DR версій політики.                                                                          | -                       | "Наступники"  | +1 рік  |
